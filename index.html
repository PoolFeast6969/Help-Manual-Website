Using the Payment Input Form

The Payment Input Form is a page of your spreadsheet that allows you to enter information into the record page in a manner that is significantly quicker the entering the information either manually into the spreadsheet, or onto a similar paper system.

Fig 2 - The Input Sheet

To operate the Input Form the user must:

  • Select the sheet Input from the tabs on the bottom left. Figure 1.
  • You can observe on this sheet, Figure 2, there are are colored areas that you can select. These colored areas are where the information is entered this text is next.
  • Any information entered into the darker green underlined areas notated the picture will be copied to the Record table when the Submit function is run.
  • The Submit function is a system that copies the information you have entered into the Input Form to the Record sheet, arranging it in a readable manner as can be seen from Figure 3. To run this function, the user must use the light green buttons on the lower right of the Input Sheet, Figure 2.
  • You may also notice a Clear function, this button allows the operator to delete the current information from the Input Form, saving time when entering similar records into the Record Sheet.
Interpreting the Stored Records

The Record Sheet is where all payment records entered into the system are stored. It takes the form of a table, with one row for each employee per week, and 26 columns that show all information that went into calculating the employees Total Pay.

Fig 3 - An Example of the Record Sheet

To operate the Record Sheet the user must:

  • Select the sheet Record from the tabs on the bottom left. Figure 1.
