Using this Manual

There are functions of Microsoft Excel that are assumed knowledge for the following Help Manual pages.

Fig 1 - The Sheet Selection Tabs

A few basic operating principals:

  • Sheets are different pages or views of the same spreadsheet file. The different functions of this Spreadsheet file have been split into 6 sheets, however, only three have to be accessed to fulfill basic operation.
  • To switch between sheets, which have names that describe their function such as Input for the "Employee Payment Form", the user can select them by clicking on the tabs as the bottom left of the Excel interface. As seen in Figure 1.
  • The cursor is the easiest method of interaction with Excel, if this Manual asks you to select or interact with some element of the screen, using the cursor is an unspoken assumption.
  • Words that appear the same in this Help Manual as they do in the spreadsheet are enclosed in quotes (""), and functions or sheets are formatted like This.
Using the Employee Payment Form

The "Employee Payment Form" is a page of your spreadsheet that allows you to enter information into the Record Sheet in a manner that is significantly quicker the entering the information either manually into the spreadsheet, or onto a similar paper system.

Fig 2 - The Input Sheet

To operate the "Employee Payment Form" the user must:

  • Select the sheet Input from the tabs on the bottom left. Figure 1.
  • You can observe on this sheet, Figure 2, there are are colored areas that you can select. These colored areas are where the information is entered.
  • Any information entered into the darker green underlined areas notated in Figure 2 will be copied to the Record table when the Submit function is run.
  • The Submit function is a system that copies the information you have entered into the "Employee Payment Form" to the Record sheet, arranging it in a readable manner as can be seen from Figure 3. To run this function, the user must use the light green buttons on the lower right of the Input Sheet, Figure 2, to submit the entry.
  • You may also notice a Clear function, this button allows the operator to delete the current information from the "Employee Payment Form", saving time when entering similar records into the Record Sheet.
Interpreting the Stored Records

The Record Sheet is where all payment records entered into the system are stored. It takes the form of a table, with one row for each employee per week, and 26 columns that show all information that went into calculating the employees Total Pay.

Fig 3 - A Part of the Record Sheet

To use the Record Sheet the user should remember to:

  • Before you begin, select the sheet Record from the tabs on the bottom left. Figure 1.
  • Notice the system of column headings and sub headings, as can be observed in Figure 3. These main headings break up the table and make it easier to find parts. Such as Task Pay and Hours Worked.
  • You may notice the colored circles in the column Total Hours, these turn red if the employee has worked more then 40 hours in a single week, and are green otherwise.
  • Another factor that may confuse the user is the absence of units in the rows. This is because the units have been moved to the column headings of the table, to make the information easier to skim through quickly.

Note: You may have to scroll left or right on the spreadsheet to see all the columns.

Using the Payment View

The Payment View function allows you to review individual entries from the Record Sheet in a less dense and more readable manner.

Fig 4 - The Recent Sheet

To operate the Recent Sheet the user must:

  • Select the sheet Recent from the tabs on the bottom left. Figure 1.
  • Notice the section titled "Payment View Select" in the bottom right of the Sheet, displayed in Figure 4 to the right. This editable area allows you to enter a Row Reference from the Record Sheet that will change the "Payment View" to display that corresponding entry.
  • To find the Row Reference of a particular wage, the user must first locate that payment in the Record Sheet. The number on the far left of the desired entry displays the Row Reference. This column is just left of the view from Figure 3.
  • In the "Payment View Select" section, the number titled "Most Recent", Seen in Figure 4, informs the user of the Row Reference of the most recent entry into the Record Sheet. This number will increment upwards as new entries are added to the system.
Using the Payment Summary

The Summary Sheet gives the user a method of easily reviewing the large amounts of information that may in the Record Sheet. Summarized in graphs and tables, as seen in Figure 5.

Fig 5 - The Summary Sheet

This Sheet does not require direct user interaction, but here are some pointers on understanding the information it provides:

  • First, select the sheet Recent from the tabs on the bottom left. Figure 1.
  • The information is sorted into two broad areas, "Time Spent on Task" lets the user see the distribution of the total hours spent working from all information in the system.
  • The section "Work Per Employee" lets the reader observe the distribution of hours on a per employee basis. Making weeding out the weakest link easier then ever.
Changing System Constants

This spreadsheet system is designed to be adaptable to changing conditions in the organization, as well as being designed so that changes in the future have no effect on calculations made in the past.

Fig 6 - The Lookup Sheet

Not all parameters can be modified by the user, but here is a list of those that can:

  • "Pay Rates" and "Bonus Rates" can be changed for future payments in the Input Sheet. In Figure 2, it can be seen that a small table on the right lists some of the factors that determine employee pay. All of the numbers in the table can be edited by the user and will take effect next time the Submit function is run. Another thing to note is that "Bonus Rates" are only applied when the hours worked on that particular task is greater then or equal to 5. This cannot be changed by the user.
  • Another parameter that can be useful to change is the list of employees currents employed. To edit this list, proceed to the Lookup Sheet, depicted in Figure 6. Editing the values under "Employees" and "Employee Phone" will make those values available when selecting workers from the Input Sheet. However, it will not update the names in the Summary Sheet.
